# Nex Development System Constitution

## Purpose
The purpose of the Nex Development System (NDS) Constitution is to define the non-negotiable governance, standards, workflows, and principles that guide all human developers and AI Developer agents building within the Nex ecosystem.

## Core Principles
1. **Repository Integrity**: Strict governance over branches, releases, and codebase consistency.
2. **Documentation First**: Comprehensive documentation and decision logging are integral parts of the product.
3. **AI Integration**: AI agents must adhere strictly to repository governance, mandatory onboarding procedures, and state handoffs.
4. **Design Excellence**: Unique, high-quality user experiences that strictly avoid generic or cliché design patterns.

## Articles

### Article 1
The main branch is protected. Never work directly on main.

### Article 2
Every task starts by creating a dedicated Git branch.

### Article 3
Every important decision must be documented.

### Article 4
Documentation is part of the product.

### Article 5
Repository language is English.

### Article 6
Conversations may be written in any language. Repository documentation must remain English.

### Article 7
Every AI must read `AI_ONBOARDING.md` before starting work.

### Article 8
Every completed task must update:
- `CURRENT_STATE.md`
- `HANDOFF.md`

### Article 9
Internationalization (i18n) is mandatory from the beginning.

### Article 10
UI design must be original. Never generic. Never based on glassmorphism. Every significant screen must explore at least three different design directions before implementation.
